# 算法统一认知框架

算法设计的统一思维工具

## 核心理念

### 算法的本质
**算法 = 在解空间上的聪明穷举 + 辅助变量优化**。核心目标：
- **无遗漏**：覆盖所有可能解
- **无冗余**：避免重复计算

### 两大基本策略
| 策略 | 核心思想 | 典型实现 |
|:---|:---|:---|
| **遍历** | 直接搜索解空间 | 迭代、递归、回溯、DFS、BFS |
| **分解** | 将解空间划分为子空间，分别求解后合并 | 分治、动态规划（带记忆化） |


## 一、解空间结构

解空间是问题所有可能解的集合，其结构由解的**数学形态**决定，而非算法名称或数据结构。

### 1. 线性解空间
**基础元素**：有序元素（数组、字符串等）

| 解的形态 | 描述 | 典型算法 |
|:---|:---|:---|
| 单点解 | 单个位置上的元素 | 顺序查找、二分查找 |
| 点对解 | 两个元素的关系 | 两数之和、三数之和、逆序对 |
| 连续段解 | 由起点和终点确定的区间 | 滑动窗口、前缀和、差分、区间最值 |
| 排列解 | 所有元素的一个全排列 | 全排列、下一个排列 |
| 子序列解 | 保持原序但不连续的元素集合 | 最长递增子序列、最长公共子序列 |

### 2. 树形解空间
**基础元素**：树节点

| 解的形态 | 描述 | 典型算法 |
|:---|:---|:---|
| 单点解 | 单个节点 | 节点查找、节点统计 |
| 路径解 | 节点序列，相邻相连 | 根到叶子路径、任意两点路径、直径 |
| 子树解 | 以某节点为根的子树（含后代） | 子树和、树形DP、树的重心 |
| 结构解 | 树的特定子结构 | 树的同构、树哈希 |

### 3. 图形解空间
**基础元素**：图节点

| 解的形态 | 描述 | 典型算法 |
|:---|:---|:---|
| 单点解 | 单个节点 | 节点度、连通分量 |
| 路径解 | 节点序列，相邻有边 | DFS/BFS路径、Dijkstra、Floyd、拓扑排序 |
| 连通分量解 | 极大连通子图 | 并查集、DFS/BFS找连通分量 |
| 生成树解 | 包含所有节点的树形子图 | 最小生成树（Prim、Kruskal） |
| 匹配解 | 无公共顶点的边集 | 二分图匹配（匈牙利） |
| 圈解 | 简单环 | 环检测、欧拉回路、哈密顿回路 |

### 4. 集合解空间
**基础元素**：无序元素

| 解的形态 | 描述 | 典型算法 |
|:---|:---|:---|
| 单元素解 | 元素的存在性 | 哈希表查找 |
| 子集解 | 原集合的一个子集 | 子集枚举、子集和、背包问题 |
| 关系解 | 两个集合的交、并、差 | 集合运算 |
| 划分解 | 将集合划分为若干组 | 并查集 |

### 5. 数域解空间
**基础元素**：数值

| 解的形态 | 描述 | 典型算法 |
|:---|:---|:---|
| 单值解 | 单个数值 | 素数判定、数值计算 |
| 数对解 | 两个数的关系 | 最大公约数、扩展欧几里得 |
| 数列解 | 一系列数值 | 素数筛、斐波那契、数位DP |
| 方程解 | 满足方程的数值 | 高斯消元、中国剩余定理 |

### 6. 复合解空间
**基础元素**：多种结构组合

| 解的形态 | 描述 | 典型算法 |
|:---|:---|:---|
| 网格路径解 | 二维网格上的路径 | 迷宫最短路径、不同路径 |
| 子矩阵解 | 矩阵中的连续子区域 | 二维前缀和、二维差分 |

**说明**  
动态规划不是独立的解空间，而是**分解策略的一种**，其解空间本质上是**具有重叠子问题的递归树**。通过记忆化（辅助变量）剪枝，将指数级遍历转化为多项式级。因此，动态规划算法分布在各个解空间类型中（线性子序列、树形子树、集合子集等），并用五要素统一描述。


## 二、遍历框架

遍历框架是遍历解空间的根本方式，由解空间结构决定。只有两种基本形式：

| 框架 | 适用场景 | 特点 |
|:---|:---|:---|
| **迭代** | 线性结构 | 显式循环控制，适合顺序访问 |
| **递归** | 树形、图形、决策树 | 隐式系统栈管理状态，适合嵌套结构 |

遍历框架决定了算法的整体流程，是算法实现的骨架。


## 三、遍历控制变量

遍历控制变量是在遍历框架内部用来指示“当前走到哪里”的变量，它驱动遍历的进程，生命周期短，随遍历不断变化。

| 遍历框架 | 典型遍历控制变量 |
|:---|:---|
| 迭代（线性） | 索引 `i`、指针 `left/right`、`slow/fast` |
| 递归（树形） | 当前节点指针 `root`、当前路径 `path`、当前行 `row` |
| 回溯 | 递归参数（路径、状态标记等） |
| 图遍历 | 当前节点编号 `u`、队列/栈中的元素 |


## 四、节点操作

节点操作是在每个被访问的节点（或子问题）上执行的具体逻辑。根据当前节点拥有的子节点（或邻居）数量，操作时机可分为不同类别。

### 1. 线性节点（单分支）
- **定义**：每个节点只有一个后续节点（链表、线性递归）
- **递归遍历**：
  - **前序**：刚进入节点，尚未处理子节点
  - **后序**：即将离开节点，已处理完唯一的子节点
- **迭代遍历**：通常只有前序操作（无法自然回溯）

### 2. 二叉树节点（双分支）
- **定义**：每个节点最多有两个子节点（二叉树、二叉搜索树）
- **可用时机**：
  - **前序**：刚进入节点，左右子节点均未处理
  - **中序**：左子节点处理完毕，右子节点尚未处理
  - **后序**：左右子节点均已处理完毕，即将离开节点

### 3. 多叉节点（多分支）
- **定义**：每个节点有多个子节点或邻居（多叉树、图DFS、回溯决策树）
- **细粒度时机**：

```cpp
进入节点（前序）
for 每个子节点 i:
    // 进入子节点 i 之前
    判断子节点合法性（剪枝）
    做选择（进入前）
    
    递归处理子节点 i
    
    // 离开子节点 i 之后
    撤销选择（离开后）
    
    // 两个子节点之间的间隙（可选）
离开节点（后序）
```

### 重要说明
- 所有时机都属于**当前节点**，只是在不同时刻执行，所获取的信息不同（后序时已获所有子节点结果，前序时仅知当前节点）
- 图形结构中的多邻居同样适用多分支节点时机，仅需额外 `visited` 标记避免重复
- 理解这些时机有助于精准控制算法行为，避免状态混乱，实现高效剪枝


## 五、辅助变量

辅助变量用于记录状态、剪枝、缓存结果，保证无冗余。根据功能分类：

| 类型 | 作用 | 示例 |
|:---|:---|:---|
| **结果存储** | 收集最终答案 | `res` |
| **状态标记** | 标记已访问/已使用 | `used`、`visited` |
| **剪枝依据** | 提供快速判断合法性的信息 | `curSum`、冲突集合 |
| **路径记录** | 存储当前部分解 | `path`、`track` |
| **临时计算** | 暂存中间结果 | `sum`、`mid` |
| **优化缓存** | 加速重复子问题 | `dp` 表、哈希表、堆 |

### 变量生命周期与存放位置
根据变量的共享范围，分为三类：

| 类型 | 描述 | 典型变量 | 存放位置 |
|:---|:---|:---|:---|
| **一次调用内** | 仅在单次函数执行中存在 | 循环索引 `i`、累加和 `sum` | 函数局部变量 |
| **递归内共享** | 在一次递归过程中共享 | 回溯中的 `path`、`used` | 递归参数（引用）或 lambda 捕获 |
| **类内长期** | 跨多次公共方法调用持久 | 并查集的 `parent`、LRU缓存 | 类的成员变量 |

**核心原则**：根据变量需要的共享范围决定存放位置，避免将递归内共享的变量错误地放在类成员中导致状态污染。


## 六、示例：全排列（回溯）

```cpp
class Solution {
    vector<vector<int>> res;      // 辅助变量：结果存储
    vector<int> path;             // 辅助变量：路径记录
    vector<bool> used;            // 辅助变量：状态标记

    void backtrack(vector<int>& nums) {
        // 节点本身的前序位置
        if (path.size() == nums.size()) {
            res.push_back(path);
            return;
        }
        
        // 多叉节点：循环处理每个子节点
        for (int i = 0; i < nums.size(); ++i) {
            // 进入子节点前（剪枝）
            if (used[i]) continue;

            // 进入子节点前（做选择）
            path.push_back(nums[i]);
            used[i] = true;

            // 递归进入子节点
            backtrack(nums);

            // 离开子节点后（撤销选择）
            path.pop_back();
            used[i] = false;
        }
        // 节点本身的后序位置（可选）
    }

public:
    vector<vector<int>> permute(vector<int>& nums) {
        used.assign(nums.size(), false);   // 初始化辅助变量
        backtrack(nums);
        return res;
    }
};
```
